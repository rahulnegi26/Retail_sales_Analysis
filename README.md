# Retail Sales Analysis using SQL

# Project Overview
This project focuses on analyzing retail sales data using SQL to uncover business insights related to sales performance, customer behavior, product categories, and time-based trends.

 # Dataset Description
The dataset contains transactional-level retail sales data with the following columns:
transaction_id – Unique ID for each transaction
sale_date – Date of sale
sale_time – Time of sale
customer_id – Unique customer identifier
gender – Customer gender
age – Customer age
category – Product category (Beauty, Clothing, Electronics)
quantity – Number of items sold
price_per_unit – Price per item
cogs – Cost of goods sold
total_sale – Total transaction value

# Objectives
Perform exploratory sales analysis using SQL
Answer real-world business questions
Practice SQL concepts like filtering, aggregation, grouping, subqueries, and window functions

# SQL Skills Used
SELECT, WHERE, ORDER BY
GROUP BY, HAVING
Aggregate functions (SUM, AVG, COUNT)
Date functions (YEAR, MONTH)
DISTINCT

# Sample Insights
Electronics category generated higher average transaction values
Afternoon shift recorded the highest number of orders
Customer purchase behavior varies significantly across categories
A small group of customers contributes a large portion of total sales
